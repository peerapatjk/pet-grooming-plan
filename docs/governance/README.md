# Governance

This folder defines the repo rules that keep `docs/`, `idea-refine/`, `spec-driven-development/`, `planning-and-task-breakdown/`, and `prototype-prompts/` from drifting into competing truths.

## Purpose

Use these artifacts when you need to answer:

- which document is authoritative for a given decision
- how work is promoted from one lane to the next
- what must be updated when scope or policy changes
- how repo-internal linking and traceability should work

## Stage

- Stage: `REPO GOVERNANCE`
- Primary audience: product, engineering, design, operations, and any contributor editing the artifact chain
- Authority boundary: these docs define repo process and authority rules; they do not replace the shared policy summary in [../launch-canon.md](../launch-canon.md) or the detailed product behavior in [../../spec-driven-development/spec.md](../../spec-driven-development/spec.md)
