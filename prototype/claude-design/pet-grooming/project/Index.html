Seven surfaces, one operating story.

This package is the full argument: design system, customer flow, merchant board, ops control room, stakeholder narrative, public landing page, and deck. Use it as a guided product proof, not a loose pile of mocks.

How it fits together

One spine, seven surfaces.

The design system sets the language. Customer, merchant, and ops express one booking from three roles. The one-pager and deck compress the same argument for review. The landing page explains the wedge publicly without creating a fake web product.

Shared
Tokens, typography, and state language

When shield, hold, review, and repeat appear in one surface, they mean the same thing everywhere else.

Cross-ref

The package is stitched so stakeholders can trace one story across public, customer, merchant, and ops views.

Boundary
The website sells the app. It does not replace it.

That constraint is enforced across the package so the launch slice stays coherent.
