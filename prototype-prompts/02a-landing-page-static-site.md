# Landing Page Static Site Prompt

## When To Use

Paste this into Claude Design to create a **static marketing landing page** for the product.

This page should direct users to download the app. It must not behave like a second booking product.

## Prompt

```text
Create a polished static marketing landing page for a Bangkok-first pet grooming booking platform.

Important constraint:
This is a static website that exists to explain the product and drive app downloads.
It must NOT include booking functionality, search functionality, onboarding functionality, or any fake web-product flow.

The goal is to help a CEO and stakeholders visualize the acquisition surface and public-facing story around the app launch.

Product context:
- Bangkok-first launch
- in-shop grooming only
- mobile app is the primary customer product
- routine services can be instantly bookable
- high-variance cases route to request-confirm
- repeat booking is a core value proposition
- payment protection exists to reduce fake bookings and no-shows
- merchants get a canonical schedule instead of fragmented LINE and phone coordination

What this landing page should do:
- communicate the product clearly
- communicate trust
- communicate the launch wedge honestly
- direct users to download the app

What it must NOT do:
- pretend booking can happen on the website
- collect onboarding information
- show fake web booking forms
- imply broad citywide or multi-service coverage beyond the launch slice

Primary audience:
- busy Bangkok pet owners who discover the product through marketing, social sharing, or merchant referral

Secondary audience:
- CEO and stakeholders reviewing the public-facing story

I want the site to include:

1. Hero section
- strong headline
- crisp subheading
- premium product visual using app screens
- primary CTA: Download the app
- secondary CTA: See how it works

2. Problem framing
- booking today is fragmented across LINE, Instagram, Facebook, and phone calls
- rebooking is too manual
- grooming availability is often unclear

3. What makes this different
- fast routine booking
- request-confirm for edge cases
- repeat booking for the same pet
- trusted schedule and reminders

4. Trust section
- explain payment protection at a high level
- explain that booking states are clear
- explain reminders and reconfirmation
- keep this calm and premium, not punitive

5. Merchant value section
- one canonical schedule
- less back-and-forth
- better no-show control

6. Launch wedge section
- Bangkok-first
- premium-leaning independent grooming shops first
- in-shop only

7. Download CTA section
- App Store / Play Store style CTAs
- strong closing value proposition

8. Footer
- simple, premium, trustworthy

Visual direction:
- premium but not luxury-brand precious
- warm, modern, and polished
- not playful-cute pet marketplace
- not generic SaaS gradient slop
- strong mobile product visuals
- elegant static marketing page with intentional typography and hierarchy

Copy direction:
- crisp
- high-trust
- no hype
- no fake "revolutionary AI" language
- clear that the app is where booking happens

Please generate:
- the full landing page
- a mobile-responsive version
- a few alternate hero directions
- one version optimized specifically for CEO and stakeholder review

Again, critical constraint:
this is a static site with app-download CTAs only.
Do not add site-based booking or onboarding functionality.
```

## Good Follow-Ups

- "Make the hero stronger and more premium."
- "Make the trust section clearer without sounding defensive."
- "Make it more obvious that booking happens in the app."
- "Tighten the launch-wedge section so it feels more disciplined."
